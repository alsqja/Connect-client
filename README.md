## 🪧 커밋 컨벤션
<details>
<summary>커밋 컨벤션 펼치기</summary>

- feat : 기능 추가

- fix : 기능 수정
  
- hotfix : 기능 급하게 수정
  
- test : 테스트 코드 작성
  
- refactor : 리팩토링
  
- docs : 문서 작업
  
- style : 코드 스타일 등 로직 변경 외 처리

- PR 은 이슈당 하나 씩
  
- 브랜치 기능별로 분리 (feature/login, feature/signup)

- 모두 approve 되면 merge
